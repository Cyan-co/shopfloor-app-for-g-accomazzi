# Solution Proposal

## Problem Statement
The current process for requesting materials on the shop floor is manual, paper-based, and inefficient. It leads to delays, errors, and a lack of visibility into the material supply chain.

## Proposed Solution
To replace the manual process with a real-time digital system. The system will provide a web-based application for Production Line Users, Warehouse Users, and Administrators to manage material requests.

## Technology Stack
- Backend: Java 21, Spring Boot 3.x
- Frontend: Angular 17+
- Database: PostgreSQL 15+
- Orchestration: Kubernetes
- Identity Provider: Keycloak
- Monitoring: Prometheus, Grafana, ELK Stack
